Introduction

The xDatabase module is a part of the Windows PowerShell Desired State Configuration (DSC) Resource Kit, which is a collection of DSC Resources produced by the PowerShell Team. This module contains the xDatabase and xDBPackage resources. xDatabase handles create/delete of a database using a dacpac or sql connection string. xDBPackage allows extraction of a dacpac or import of a bacpac from a database.

All of the resources in the DSC Resource Kit are provided AS IS, and are not supported through any Microsoft standard support program or service. The ""x" in xDatabase stands for experimental, which means that these resources will be fix forward and monitored by the module owner(s).

Please leave comments, feature requests, and bug reports in the Q & A tab for this module.

If you would like to modify xDatabase module, feel free. When modifying, please update the module name, resource friendly name, and MOF class name (instructions below). As specified in the license, you may copy or modify this resource as long as they are used on the Windows Platform.

For more information about Windows PowerShell Desired State Configuration, check out the blog posts on the PowerShell Blog (this is a good starting point). There are also great community resources, such as PowerShell.org, or PowerShell Magazine. For more information on the DSC Resource Kit, check out this blog post.

Installation

To install xDatabase module

  • Unzip the content under $env:ProgramFiles\WindowsPowerShell\Modules folder

To confirm installation:

  • Run Get-DSCResource to see that xDatabase and xDBpackage are among the DSC Resources listed

Requirements

This module requires the latest version of PowerShell (v4.0, which ships in Windows 8.1 or Windows Server 2012 R2). To easily use PowerShell 4.0 on older operating systems, install WMF 4.0. Please read the installation instructions that are present on both the download page and the release notes for WMF 4.0.

Description

This xDatabase module contains the xDatabase and xDBPackage resources. xDatabase handles create/delete of a database using a dacpac or sql connection string. xDBPackage allows extraction of a dacpac or import of a bacpac from a database. For information on Data-Tier Applications please refer to http://technet.microsoft.com/en-us/library/ee240739(v=sql.105).aspx

 

Details

xDatabase resource has the following properties:

  • Credentials: The credentials to connect to the sql server
  • SqlServer : The sql server to connect.
  • SqlServerVersion: The version of the Sql Server, this property can take the following values :

2008-R2, 2012 or 2014

  • BacPacPath : The path to the .bacpac file to be used for database restore. If this is specified the DacPacPath(see below) cannot be specified. This is an optional parameter
  • DacPacPath: The path to the .dacpac file to be used for database schema deploy. If this is specified the BacPacPath(see below) cannot be specified.This is an optional parameter.
  • DacPacApplicationName: For deploying a database using .dacpac file, an application name is to be given with which the dacpac is registered. This is needed to support database upgrade using .dacpac file. This must specified if DacPacPath is provided.
  • DacPacApplicationVersion: This is an optional parameter needed for registration for Database deployment using .dacpac file for dacpac registration. If not specified the version
  • DatabaseName: The name of the Database to be deployed.

xDBPackage resource has the following properties:

  • DatabaseName: The name of the Database to be deploy
  • SqlServer : The sql server to connect.
  • SqlServerVersion: The version of the Sql Server, this property can take the following values :

2008-R2, 2012 or 2014

  • Path : The path to the .bacpac or .dacpac file to be used to export a database to a bacpac or extract a db to a dacpac respectively.
  • Type: This property can take DACPAC and BACPAC as allowed values to specify the action to be taken i.e. DACPAC for dacpac extraction and BACPAC for export to a bacpac for the given database.

 

Renaming Requirements

When making changes to these resources, we suggest the following practice:

1.     Update the following names by prefixing your company/community name and replacing the "x" with "c" (short for "Community") or another prefix of your choice:

o    Module name (ex: xDatabase becomes cDatabase)

o    Resource folder (ex: xDatbase becomes Contoso_cDatbase)

o    Resource Name (ex: xDatbase becomes Contoso_cDatabase)

2.     Update module and metadata information in the module manifest

3.     Update any configuration that use these resources

We reserve resource and module names without prefixes ("x" or "c") for future use (e.g. "xDatabase"). If the next version of Windows Server ships with a "Database" resource, we don't want to break any configurations that use any community modifications. Please keep a prefix such as "c" on all community modifications.

Example: Deploy Database using DacPac

This configuration will deploy the database with the schema specified in the dacpac. If the db exists, the new schema will be deployed since dacpac deploy is idempotent.

 

configuration DacDeploy

{

param

(

[Parameter(Mandatory)]

[ValidateNotNullOrEmpty()]

[String]$Ensure,

 

[Parameter(Mandatory)]

[ValidateNotNullOrEmpty()]

[String]$DatabaseName,

 

[Parameter(Mandatory)]

[ValidateNotNullOrEmpty()]

[String]$SqlServer,

 

[Parameter(Mandatory)]

[ValidateNotNullOrEmpty()]

[String]$SqlServerVersion,

 

[String]$DacPacPath,

 

[String]$BacPacPath,

 

[Parameter(Mandatory)]

[ValidateNotNullOrEmpty()]

[String]$NodeName,

 

[PSCredential]

$Credentials,

 

[string]$DacPacApplicationName

 

)

Node NodeName

{

xDatabase DeployDac

{

Ensure = $Ensure

SqlServer = $SqlServer

SqlServerVersion = $SqlServerVersion

DatabaseName = $DatabaseName

Credentials = $Credentials

DacPacPath = $DacPacPath

DacPacApplicationName = $DacPacApplicationName

}

}

}
